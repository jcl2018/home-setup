# Codex Projects

Use this directory as the durable state layer for Codex work.

- One project per slug under `~/.codex/projects/<project-slug>/`
- `project.md` is the durable summary
- `designs/`, `reviews/`, `qa/`, `ship/`, and `retro/` hold stage outputs
- Use `~/.codex/bin/codex-project-log ensure` before writing artifacts
