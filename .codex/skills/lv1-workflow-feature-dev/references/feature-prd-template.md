# Feature PRD Template

Use this template for per-item feature PRDs. The default path is `<repo-knowledge-root>/features/<slug>.md` unless the repo already documents another convention.

## Title

- Title:
- Slug:
- Status: proposed | planned | in_progress | blocked | done

## Summary

- One short paragraph describing the feature item and the intended outcome.

## Problem

- What is broken, missing, slow, confusing, or strategically important?
- Why should this item exist now?

## Users Or Workflows Affected

- Who uses this?
- Which workflow, job, or surface changes?

## Scope

### In Scope

- The concrete behaviors, screens, endpoints, jobs, or docs this item will change.

### Out Of Scope

- The related ideas that are intentionally excluded from this item.

## Requirements

### Functional Requirements

- The observable behaviors the implementation must provide.

### Non-Functional Requirements

- Performance, reliability, security, accessibility, maintainability, or operational expectations when relevant.

## UX, API, Data, Or Migration Notes

- Include only the sections that matter for this item.

## Dependencies

- Upstream prerequisites, sequencing constraints, or linked feature items.

## Risks And Unknowns

- What could derail the work or force a design change?

## Acceptance Criteria

- The conditions that make this item done.

## Verification Plan

- The smallest commands, tests, or checks that should prove the item works.

## Related Sources

- Repo docs, source files, design notes, issues, or prior PRDs.

## Last Updated

- YYYY-MM-DD
